- 银行融资 -

此合约在区块链中为企业提供直接向对应银行申请融资操作，同时银行也可以通过区块链进行企业的融资申请处理。

最后银行融资记录都会保存在区块数据中永远不会被修改和删除。

if 企业向对应银行发出融资申请；than 系统将申请发送到对应银行的服务器

if 银行同意融资申请；than 企业获得融资

if 银行不同意融资申请；than 企业无法获得融资

- 金融交易 -

此合约在区块链中为用户直接向证券商进行投资金融产品操作，证券商作为中间人经过快速审核撮合交易。

最后金融交易记录会保存在区块数据中永远不会被修改和删除。

if 用户向对应证券商发出金融交易申请；than 系统将申请发送到对应证券商的服务器

if 证券商确认用户为自家客户；than 证券商将交易信息汇集到交易市场进行交易

if 证券商确认该交易为非法交易；than 证券商拒绝该交易进行

- 保险买卖 -

此合约在区块链中为用户直接向保险商进行购买保险的操作，保险商会对用户的信息进行审核和评估最后签约。

最后保险买卖记录都会保存在区块数据中永远不会被修改和删除。

if 用户向对应保险商发出保险购买申请；than 系统将申请发送到对应保险商的服务器

if 保险商在一定时间内审核和评估后同意订单；than 保险商确认该保险签约生效

if 保险商审核和评估后不同意该订单；than 保险商拒绝此次保险订单的签约

- 票据流转 -

此合约在区块链中为企业直接向企业或银行进行票据流转操作，开立的票据可以被背书、转让或贴现。

最后票据流转记录会保存在区块数据中永远不会被修改和删除。

if 企业向对应接收方发出票据流转申请；than 系统将申请发送到对应接收方的服务器

if 接收方在指定时间内对票据进行审核并同意；than 接收方确认接收该票据并获得所属权

if 接收方拒绝了该票据的流转；than 票据申请被返回到企业的服务器

- 债券发行 -

此合约在区块链中为企业直接向银行或证券商进行债券发行操作，银行或证券商对该证券负责并承销发行。

最后债券发行记录会保存在区块数据中永远不会被修改和删除。

if 企业向对应银行或证券商发出债券发行申请；than 系统将申请发送到承销商的服务器

if 承销商审核企业债券的信用度并同意；than 承销商同意该债券的承销并发行该债券

if 承销商审核该债券为高风险并拒绝；than 债券发行申请被拒绝系统将此债券的状态注销
